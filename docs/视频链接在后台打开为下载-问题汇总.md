# 后台管理「查看视频」链接打开为下载（非直接播放）— 问题汇总

## 一、问题现象（用户侧）

1. 在后台管理详情中点击「查看视频」时，浏览器会**直接下载**视频文件，而不是在当前页或新标签页中**直接播放**。
2. 将详情中显示的 Request URL（OSS 视频地址）复制到浏览器地址栏打开，同样会触发**下载**，无法在页面内播放。
3. 网络面板中该视频请求状态为 **206 Partial Content**（无 403），下载后的本地文件可以正常播放。

## 二、原因分析

### 2.1 技术原因

- 视频文件存储在 **阿里云 OSS**，访问 URL 形如：  
  `https://<bucket>.oss-cn-<region>.aliyuncs.com/uploads/<id>.mp4`
- 通过命令行查看响应头可知，OSS 对该对象的返回中包含：
  - **Content-Type: video/mp4**（类型正确）
  - **Content-Disposition: attachment**（表示“作为附件下载”）
- 浏览器根据 **Content-Disposition: attachment** 将响应视为“下载文件”，因此会触发下载，而不会在页面内内联播放。这是符合 HTTP 语义的预期行为。

### 2.2 与小程序播放的关系

- **小程序端**使用 `<video>` 等组件请求同一 OSS URL 时，是**流式请求**，不依赖“导航到链接”的行为，因此**不受 Content-Disposition: attachment 的同样影响**，可以正常播放。
- 因此：「后台点击链接会下载」与「小程序能否播放」是两类场景；后台表现为下载**不影响**审核时对内容的查看（下载后仍可播放），也不影响小程序端用户正常观看。

## 三、如何自行验证（供后续同类问题参考）

### 3.1 查看 OSS 响应头（确认是否 attachment）

在 **PowerShell** 中（项目目录或任意目录均可）执行：

```powershell
curl.exe -I "https://<你的OSS桶域名>/uploads/<对象key>.mp4"
```

关注响应头中的：

- **Content-Type**：通常为 `video/mp4`。
- **Content-Disposition**：若为 `attachment` 或 `attachment; filename=...`，则浏览器会触发下载；若为 `inline` 或未设置，则浏览器可能内联播放。

注意：在 PowerShell 中请使用 `curl.exe`，避免使用内置别名导致行为不同。

### 3.2 在浏览器开发者工具中查看响应头

1. **先**打开开发者工具（F12）并切到「网络 / Network」，筛选选「全部」。
2. **再**在地址栏输入或粘贴 OSS 视频 URL 并回车。
3. 在请求列表中选中该 `.mp4` 请求，右侧「标头 / Headers」→「Response Headers」中可查看 Content-Type、Content-Disposition。

若当前页为“下载”场景，部分浏览器可能只显示“临时标题”，响应头不全，此时以 `curl.exe -I` 为准。

## 四、产品结论与可选方案

### 4.1 当前结论（本项目）

- **后台管理中点击「查看视频」变为下载、复制 URL 到浏览器也变为下载**，是可接受的行为。
- 原因：不影响审核流程（下载后即可查看内容），且**不影响小程序端用户正常播放**。
- 因此**无需**为“后台内联播放”做 OSS 元数据或前端改动。

### 4.2 若未来需要“在浏览器中直接播放”

可在以下两种方向中择一或组合使用：

1. **单对象修改**：在阿里云 OSS 控制台找到该对象 → 设置元信息 → 将 **Content-Disposition** 删除或改为 `inline`（可选 `inline; filename=xxx.mp4`），保存后浏览器打开该 URL 即可内联播放。
2. **新上传视频统一为内联播放**：在上传至 OSS 时（无论直传还是经后端代理），在请求中设置对象元数据：
   - **Content-Type**: `video/mp4`
   - **Content-Disposition**: `inline`（或不设置）
   具体实现方式取决于当前上传链路（OSS PostObject 表单项 / 后端 SDK 上传时的 headers）。

## 五、状态码说明（避免误判）

- **206 Partial Content**：浏览器/播放器对音视频常使用 Range 请求，服务器返回 206 为**正常**，表示按范围返回内容，并非错误。
- **403**：表示无权限访问；若未出现 403，说明当前 OSS 权限配置允许访问该对象。
