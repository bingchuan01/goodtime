# 问题解决记录

## 真机详情页首次加载失败（2025-03）

### 问题描述

- **现象**：真机点击项目卡片后，先出现「加载失败」页，需返回再点一次才能看见详情
- **环境**：开发者工具正常，真机调试/体验版复现
- **用户反馈**：点击两次返回才能看见详情页

### 根因分析

详情页首次进入时发起 API 请求，真机环境下网络连接可能尚未就绪，导致首次请求失败。

### 解决方案：首页预取

**思路**：在已建立连接的首页发起详情请求，成功后再跳转，避免在新页面首次请求。

**实现要点**：

1. **app.js**：增加 `globalData.prefetchedProject` 用于传递预取数据
2. **pages/index/index.js**：点击卡片时先请求详情，成功后再 `wx.navigateTo`
3. **pages/video-detail/video-detail.js**：`onLoad` 优先检查 `prefetchedProject`，有则直接使用，无则调用 `loadProjectDetail`

**相关修改**：

- `showContent` + 350ms 延迟：避免过渡动画期间底部详情图闪现
- `applyProjectData`：统一处理项目数据的应用逻辑

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| app.js | globalData.prefetchedProject |
| pages/index/index.js | onProjectTap 改为先请求再跳转 |
| pages/video-detail/video-detail.js | 预取数据读取、showContent、applyProjectData |

### 回滚方式

回退到修复前的状态（标签 fix-detail-prefetch 的上一提交）：

```bash

```

或回退整个项目到修复前：

```bash
git log --oneline -3  # 查看提交历史

```

## 优先方案：真机首次请求失败类问题

**遇到「真机首次请求失败、二次操作成功」时，优先考虑：**

1. **预取方案**：在上一页（已建立连接的页面）发起请求，再跳转
2. 避免在新页面 `onLoad` 中直接发起首次关键请求
